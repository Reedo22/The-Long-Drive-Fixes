# TLD Public MP — plugin manifest
# Format: <PluginAssemblyName>  <Version>  <DownloadURL>
# Lines starting with # are comments. Blank lines are skipped.
#
# The TLDPubMPUpdater patcher fetches this file each launch, compares <Version>
# against the BepInPlugin version of each local DLL, and downloads anything
# newer as *.dll.update (applied on next launch).
#
# IMPORTANT: every version below MUST match the actual AssemblyVersion of the
# DLL in /public at the same URL. Mismatch = updater downloads a wrong-versioned
# DLL believing it's an upgrade, and your install rolls forward (or backward)
# to whatever's at the URL. This is what caused the "weird issues" after the
# v2.x experiments — the manifest still claimed v2.11 when we'd reverted local
# installs to the v1.1 baseline, so the updater quietly re-shipped v2.11 each
# launch.
#
# Dev-only plugins (TLDPubBridge, TLDPubFakeId, TLDPubLoopback's experimental
# matchmaking patches, TLDPubMPCapture, TLDPubFullLog, TLDPubSpawner,
# TLDPubItemSmooth) are intentionally NOT listed — they're not part of the
# public release, and not listing them means the updater leaves whatever's
# installed alone instead of trying to upgrade them.
